Week 3

 

Now that we have Git configured, started tracking some files for version control, had an introduction to SSH and how it works with remote repositories, it’s time to create our remote repositories on GitHub. Along with creating our remote repo’s for our course work, we’re also going to see how GitHub will be integral for our CI/CD build pipeline.

 

To continue with this week, please ensure that you’ve been completing the exercises and lab work. We’re going to look at how SSH is configured & used on the command line and continue working with our CLI text editors.

 

The following requisites are required to complete this week's lab:

 

  1. Git has been configured in your local environment with your username and email that you would like linked to your GitHub profile. Your GBC email is recommended for integration so you can take advantage of the Education Pack they offer if you have not yet applied
  2. You’re familiar with nano, vi or vim. If you aren’t, remember you can always use the `man` command we previously used to pull up the editor's documentation
  3. Access to your BASH shell on Windows, macOS or Linux
